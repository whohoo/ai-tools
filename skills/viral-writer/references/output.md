# 文章输出格式要求

## 一篇文章的输出路径及格式

### 最终的输出路径和文件名规则如下

- 路径：项目根目录下的 `articles/[YYYY-MM]/`
- 文件名：`[主题简称].md`
- 文件名包含大小写, 标题是核心, 中文标题时，不要使用urlencode, 用回中文即可, 不可包含不符合文件名命名规范的字符及emoji表情
- [YYYY-MM], [主题简称] 分别替换为当前年月和文章标题，[主题简称] 中的特殊字符需要替换为下划线。

### 一篇文章包含以下三部分

1. 元数据, 放在文件的开头，使用 yaml frontmatter 格式
2. 摘要, 放在元数据的后面, 使用>符号引用，不超过128字符
3. 正文

## 文章格式要求

1. 必须是 markdown 格式
2. 正文结束后**不要**再插入其它数据, 比如生成图片建议等文字, 生成图片建议写在元数据中
3. 格式如下

```markdown
---
theme: default
themeName: 默认主题
title: [最终选用的标题]
titleOptions: 
  - [最终选用的标题]
  - [备选标题1]
  - [备选标题2]
  - [备选标题3]
  - [备选标题4]
imagePrompt:
  coverImage: [封面图prompt]
  bodyImage: 
    - [正文配图prompt1]
    - [正文配图prompt2]
    - [正文配图prompt3]
    - [正文配图prompt4]
  bodyImagePosition:
    - [正文配图1建议插图位置]
    - [正文配图2建议插图位置]
    - [正文配图3建议插图位置]
    - [正文配图4建议插图位置]
---

> [本篇文章的摘要]

[正文内容]
```
